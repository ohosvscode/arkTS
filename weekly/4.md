# 实习周记 第四周（2月2日 - 2月6日）

本周主要围绕 ArkTS 调试器相关工作进行，创建了仓库 ohosvscode/debugger，在 ArkTS 插件中接入该仓库的调试适配器，并梳理、完善了该仓库中的断点、变量查看、暂停状态及 CI 发布等能力。其实该仓库做的实际上就是把鸿蒙 DevEco Studio 中类似Chrome Debugger Adapter（CDP）的协议实现转为 VS Code的Debugger Adapter（DAP）协议实现。

在 ohosvscode/debugger 仓库中，我基本完成了鸿蒙 ArkTS 应用的断点调试功能的相关开发：实现并完善 breakpoints 与 breakpoint-store，支持在 launch 时下发断点位置、在调试会话中同步断点状态；更新 launch-request 与 debug-session，使调试启动流程与 HAP 安装、设备类型等配置一致，并配合 test/app 的 build-profile、launch.json 做联调。

在变量查看方面，新增 variable 与 variable-store、paused-state 等数据结构，在调试暂停时展示作用域与变量；在 adapter 与 debugger-adapter 中对接变量与暂停状态请求，支持「单步」「继续」等调试操作。此外在仓库中和其他仓库一样，统一接入 changeset 与 GitHub Actions 的 publish 流程，更新 README 与使用说明，补充开发环境准备、hdc/hvigor 使用方式及 usage 示例与 gif，便于后续维护与协作。然后，我将该仓库发布到了 npm 上，包名为 @arkts/debugger，可供所有开源贡献者使用。

同时，我本身在 ArkTS 插件侧也直接依赖了 @arkts/debugger 包，并在 src/debugger/ 中注册 ets 类型的 DebugAdapterDescriptor，导入 @arkts/debugger 包的适配器实现，使开发者可在 VS Code 中直接对 ArkTS 应用进行断点调试与变量查看。
